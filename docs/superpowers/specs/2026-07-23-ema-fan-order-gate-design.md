# Гейт «EMA раскрыты» для контр-сигнала

## Контекст

Реальный пример SYNUSDT: после резкого вертикального пампа (0.03 → ~1.0) и обратного отката бот выдал бы контр-сигнал, хотя структура ещё не устаканилась — недельная EMA7 (0.2126) оказалась ВЫШЕ EMA14 (0.1712), то есть короткая средняя обогнала длинную из-за одной экстремальной свечи. На правильном примере (ALICE, реальный сигнал бота) EMA7/14/28 идут строго по возрастанию периода — короткая ближе к цене, длинная дальше, без пересечений («раскрыты веером»). Разница видна глазами на графике и имеет точный числовой след — инверсия порядка EMA.

## Критерий

Для валидности контр-сигнала прогретые EMA (`emas`, уже отфильтрованы `EMA_WARMUP_FACTOR`) должны идти строго монотонно по периоду:

- **short** (резистенс сверху, откат снизу-вверх): `EMA7 < EMA14 < EMA28` — короткая ближе к цене, длинная дальше
- **long** (поддержка снизу, откат сверху-вниз): `EMA7 > EMA14 > EMA28` — зеркально

Если прогрета только одна EMA — проверять нечего, гейт пропускается (как и раньше). Если прогрето две и больше, но порядок нарушен (перепутаны/пересеклись) — сигнал не шлём, структура ещё не сформировалась после недавнего резкого движения.

## Реализация

Новая чистая функция в `ema_pullback.py`:

```python
def emas_fanned(emas: dict[int, float], counter_direction: str) -> bool:
    values = [emas[p] for p in sorted(emas)]
    if counter_direction == "short":
        return all(a < b for a, b in zip(values, values[1:]))
    return all(a > b for a, b in zip(values, values[1:]))
```

В `build_pullback_signal`: гейт сразу после `if not side: return None` (до выбора `entry_period`) — `if not emas_fanned(emas, counter_direction): return None`. Проверяется на полном наборе прогретых `emas`, не только на подмножестве `side` — важна структура целиком, а не только те EMA, что лежат по нужную сторону от цены.

## Тестирование

- `test_ema_pullback.py`: новый тест на сценарии SYN (EMA7 > EMA14 при counter_direction="short" → `None`) и на существующие проходящие сценарии (ALICE-подобный порядок → сигнал как раньше). Проверить, что кейс с одной прогретой EMA (HEMI и подобные) не ломается.
- Существующие 40/40 тестов должны остаться зелёными без изменений (кроме, возможно, синтетических фикстур, где EMA специально не в порядке — если такие есть, разобрать отдельно).

## Вне рамок

- Не добавляем отдельную проверку «последняя недельная свеча не экстремальная» — инверсия порядка EMA уже ловит именно этот случай на реальном примере. Если на практике всплывёт кейс, где порядок формально верный, а сигнал всё равно плохой из-за свежего движения — добавим отдельным гейтом позже.
- Не меняем выбор `entry_period`, расчёт стопа/тейка, санитарный потолок дистанции — только добавляем гейт перед существующей логикой.
