# Гейт «свежий спайк» для контр-сигнала

## Контекст

Реальный пример SYNUSDT: резкий вертикальный памп (0.03 → ~1.0) и обратный обвал случились совсем недавно (в пределах 1-2 недельных свечей). Бот выдал бы по нему контр-сигнал, хотя структура ещё не устаканилась — цена ещё не «улеглась» после разового экстремального движения. На правильном примере (ALICE, реальный сигнал бота) вход находится после долгого плавного тренда и последующего бокового затишья — никакого свежего вертикального движения нет.

Изначально рассматривался гейт на порядок EMA7/14/28 («раскрыты веером»), но численная проверка на синтетических данных показала: и валидный тест `test_build_pullback_signal_short_counter_to_pump` (плавный аптренд), и SYN-подобный сценарий (долгий даунтренд + один резкий шип) дают ОДИНАКОВЫЙ порядок EMA (EMA7 > EMA14 > EMA28) — порядок EMA не различает "плавный тренд" от "точечный спайк внутри тренда". Гейт на порядок EMA отклонён.

## Критерий

Смотрим на последние `SPIKE_LOOKBACK_CANDLES` (2) недельных свечи. Для каждой считаем «тело» в процентах:

```
body_pct = abs(close - open) / open * 100
```

Если тело любой из этих свечей превышает `SPIKE_BODY_PCT_THRESHOLD` (50.0) — считаем, что было свежее экстремальное движение (сам памп/дамп или немедленный откат от него), структура ещё не устаканилась, сигнал не шлём.

Смотрим именно на последние 2 свечи (не только самую последнюю), потому что экстремальным телом может быть как сама свеча пампа, так и следующая за ней свеча отката — обе указывают на "свежесть" события. Если экстремальное движение было 3+ недели назад — гейт не срабатывает, сигнал проходит как обычно (это про свежесть, не постоянный бан пары).

## Реализация

Новая чистая функция в `ema_pullback.py`:

```python
SPIKE_LOOKBACK_CANDLES = 2
SPIKE_BODY_PCT_THRESHOLD = 50.0


def has_recent_spike(weekly_candles: list[dict]) -> bool:
    recent = weekly_candles[-SPIKE_LOOKBACK_CANDLES:]
    return any(
        abs(c["close"] - c["open"]) / c["open"] * 100 > SPIKE_BODY_PCT_THRESHOLD
        for c in recent
    )
```

В `build_pullback_signal`: гейт в самом начале, до расчёта EMA — `if has_recent_spike(weekly_candles): return None`.

Свечи в `weekly_candles` уже содержат поле `open` (приходит из `get_klines()` в `crypto_bot.py`, используется как есть) — самой функции менять не нужно. Нужно обновить тестовый хелпер `make_weekly_candles()` в `tests/test_ema_pullback.py`, чтобы он тоже выставлял `open` (сейчас там только `close`/`high`/`low`) — иначе `has_recent_spike` упадёт на `KeyError`.

## Тестирование

- `test_ema_pullback.py`: новый тест на SYN-подобный сценарий (последняя свеча с телом > 50% → `None`), новый тест на свечу пампа не в последних двух позициях (3+ недели назад → сигнал проходит как обычно). Обновить `make_weekly_candles()` — добавить `open` с дефолтом, который не создаёт ложного спайка (например, равным `close`, если явно не переопределён).
- Существующие 40/40 тестов должны остаться зелёными после обновления хелпера (дефолтный `open == close` даёт `body_pct == 0`, гейт не сработает).

## Вне рамок

- Не меняем выбор `entry_period`, расчёт стопа/тейка, санитарный потолок дистанции — только добавляем гейт перед существующей логикой.
- Не переиспользуем `PUMP_PCT`/`STRONG_PUMP_MULTIPLIER` из `crypto_bot.py` — это внутридневные пороги для другого гейта (IIV), здесь свой недельный порог с собственным смыслом (свежесть спайка, не сила движения).
